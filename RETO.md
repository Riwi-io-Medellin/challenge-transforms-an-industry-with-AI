# El Reto: Speed Run de 30 Minutos

## Objetivo

Crear un prototipo funcional de una interfaz moderna potenciada por IA que resuelva un problema real de una de las tres industrias presentadas.

## Restricciones

- **Tiempo:** 30 minutos (estricto)
- **Herramienta:** [Lovable.dev](https://lovable.dev) (o similar)
- **Modalidad:** Equipos de 3-6 personas
- **Entregable:** Deploy en vivo + Repositorio GitHub

---

## Paso 0: Selección del Brief (Minuto 0-2)

Elige **UNO** de estos tres escenarios:

### **Observación**
Eres libre de escoger un caso fuera de los siguientes escenarios, lo importante es explicar la transformación de la industria con la IA.

### **Opción A: Logística**
**"Dashboard de Control de Envíos Inteligente"**

Crea una pantalla que muestre:
- Mapa interactivo con rutas en tiempo real
- Alertas de IA: retrasos, clima adverso, paquetes dañados
- Predicciones de entrega con nivel de confianza
- Análisis visual de fotos de paquetes

---

### **Opción B: Legal**
**"Analizador de Contratos Inteligente"**

Crea una interfaz donde:
- Subes un PDF de contrato
- Muestra semáforo de riesgos (Rojo/Amarillo/Verde)
- Resalta cláusulas peligrosas automáticamente
- Chat para preguntar sobre el contrato

---

### **Opción C: Retail**
**"Personal Shopper AI"**

Crea un chat visual donde:
- Usuario describe su necesidad en lenguaje natural
- IA muestra combinaciones de ropa/productos
- Explica por qué recomienda cada item
- Carrito inteligente con sugerencias contextuales

## Entregables

### Checklist Final

Antes de presentar, verifica que tienes:

- [ ] **Repositorio GitHub** público con el código
- [ ] **URL de deploy** funcionando (desde Lovable)
- [ ] **README.md** en el repo con:
  - Descripción del proyecto
  - Screenshots o GIF de la interfaz
  - Instrucciones de instalación (si aplica)
- [ ] **Pitch preparado** (3 minutos máximo)
- [ ] **Demo funcional** (sin errores críticos)

### Formato de Entrega

Completa este formulario (o comparte en el chat):

```markdown
## Entrega - [Nombre del Equipo]

**Industria elegida:** [Logística / Legal / Retail]

**Repositorio GitHub:** https://github.com/tu-usuario/tu-repo

**URL del Deploy:** https://tu-proyecto.lovable.app

**Integrantes:**
- Nombre 1 - Rol
- Nombre 2 - Rol

**Descripción breve (50 palabras):**
[Tu descripción aquí]
```

## Reglas del Juego

### Permitido
- Usar cualquier prompt que se te ocurra
- Iterar cuantas veces quieras con la IA
- Buscar inspiración en internet
- Pedir ayuda a los instructores sobre el uso de Lovable

### No Permitido
- Copiar código de otros equipos
- Usar código pre-escrito antes del reto
- Exceder el tiempo límite (30 minutos)
- Presentar sin repositorio GitHub

---

## Preparación del Pitch (Minuto 27-30)

Define quién habla (si es equipo). El pitch debe responder:

1. **¿Qué dolor solucionan?** (30 segundos)
   - Describe el problema de la industria elegida
   
2. **¿Cómo la IA cambia el proceso?** (60 segundos)
   - Conecta con los casos de uso vistos
   - Explica qué hace "inteligente" tu interfaz
   
3. **Demo en vivo** (60 segundos)
   - Mostrar la URL generada
   - Navegar por las funcionalidades clave

**Tiempo total:** 3 minutos máximo.
